## What Problem Does This Solve?

### The Problem
Crypto alpha is fleeting. By the time you manually find a profitable wallet, the opportunity is gone. Human traders can't monitor 1,000+ wallets 24/7, detect coordinated whale movements, or adapt strategies in real-time.

### The Solution
HydraNet is a **self-evolving AI system** that autonomously tracks wallets, clusters entities, detects alpha patterns, and generates trade signals — then improves its own agents through evolutionary pressure.

### Who Uses This?
1. **Crypto traders** — automated alpha detection and trade signals
2. **Fund managers** — wallet intelligence and risk monitoring
3. **Researchers** — on-chain forensics and entity clustering

## Multi-Head Decision Architecture

HydraNet uses a **multi-head controller** where each head specializes in one domain:

```
                    ┌── Market Head ──────┐
Normalized Data ───→├── Trading Head ─────┼──→ Controller ──→ Final Decision
(from pipeline)     ├── Risk Head ────────┘    (weighted     (with confidence
                    └── On-Chain Head           fusion)       + reasoning)
```

| Head | Weight | Role |
|------|--------|------|
| **Market Analysis** | 1.0x | Market regime, trend, opportunity scoring |
| **Trading Decision** | 1.5x | Entry/exit plans, position sizing |
| **Risk Management** | 2.0x | Risk assessment, veto power, capital protection |
| **On-Chain Intel** | 1.2x | Whale moves, rug warnings, accumulation patterns |

**Risk veto**: If the Risk Head says "avoid" with >30% confidence, it overrides all other heads. Safety first.

## Pipeline: Dataset → Train → Evaluate

Unlike traditional ML (gradient descent), HydraNet uses **evolutionary prompt optimization**:

```
1. DATASET    — Build labeled data from historical transactions
                (wallet scoring: profitable/unprofitable)
                (signal quality: alpha/noise)

2. EVALUATE   — Run agent prompts against test data, measure accuracy

3. IMPROVE    — LLM analyzes errors, generates improved prompts

4. REPEAT     — Until accuracy ≥ 90% or max iterations reached
```

This is measurable, reproducible, and improvable — not just "prompting."

## Evaluation Metrics

HydraNet tracks these metrics continuously:

| Metric | What it measures | Target |
|--------|-----------------|--------|
| `accuracy` | Agent prediction correctness | >70% |
| `latency_ms` | Decision response time | <2000ms |
| `cost_usd` | LLM API cost per decision | <$0.01 |
| `signal_roi` | ROI of generated signals | >0% |
| `win_rate` | % of profitable trades | >55% |
| `tx_per_second` | Data ingestion throughput | >100 |
| `agent_composite_score` | Weighted overall score | >0.5 |

All metrics available at `GET /metrics` with trend analysis.

## Philosophy

> Stop prompting AI. Let AI evolve itself.

HydraNet is not a chatbot wrapper. It's a **self-improving intelligence system** that gets smarter through Darwinian selection — the same force that drives biological evolution.

The multi-head architecture ensures no single point of failure in decision-making. Risk management has veto power. Every decision is traceable, measurable, and improvable.
