# HydraNet

### 3 AI enter. 1 AI leaves.

HydraNet lets multiple LLMs **fight**, **critique**, and **evolve** — until only the best answer survives.

> **This project was generated by GPT (OpenAI)** as part of an AI coding comparison.
> See also: [Claude's version (CodeSnap)](https://github.com/q15432123/codesnap) | [Gemini's version (AutoLogic)](https://github.com/q15432123/AutoLogic)

---

## What is this?

You give HydraNet a task. It sends it to **3 different AI models simultaneously**. They each generate an answer, then **tear apart each other's work**. An AI judge scores them. Then HydraNet **evolves the ultimate answer** by combining the best elements from all three.

The result is consistently **better than any single AI** could produce alone.

```
You: "Write a rate limiter using token bucket algorithm"

GPT-4o  → writes solution A
Claude  → writes solution B
Gemini  → writes solution C

GPT-4o attacks Claude: "Your implementation isn't thread-safe"
Claude attacks Gemini: "O(n) space is wasteful here"
Gemini attacks GPT-4o: "Missing edge case for burst handling"

Judge: Claude wins (88/100)

HydraNet Evolved Answer: combines thread safety (from Claude's approach)
+ memory efficiency (from GPT) + burst handling (from Gemini's critique)
= strictly better than all 3
```

## Core Features

### Prompt Battle
3 AIs answer the same question. They don't know what the others wrote. Pure independent thinking.

### Cross-Critique
Each AI reads the other two answers and **attacks weaknesses**. No mercy. Blind spots get exposed.

### Judge System
An impartial AI judge scores every response on:
- **Correctness** (40%) — is it right?
- **Completeness** (25%) — did it cover everything?
- **Clarity** (20%) — is it easy to understand?
- **Creativity** (15%) — is the approach clever?

### Evolution Mode
The killer feature. HydraNet takes ALL responses + ALL critiques and synthesizes an answer that's **strictly better than any individual response**. This is where the magic happens.

### Self-Improving Prompts
Feed a prompt through multiple evolution rounds. Each round generates variations, tests them, and synthesizes the best. Your prompt gets dramatically better.

### ELO Leaderboard
Track which AI wins most often. See real performance data across hundreds of battles.

## Why This is Different

**ChatGPT wrapper?** No.
**Another agent framework?** No.
**LangChain but different?** No.

HydraNet is an **adversarial intelligence system**. It uses competition and critique — the same forces that drive scientific peer review — to produce outputs that no single AI can match.

The insight is simple: **AIs are better at finding flaws in others' work than avoiding flaws in their own.** HydraNet exploits this asymmetry.

## Quick Start

```bash
git clone https://github.com/q15432123/HydraNet.git
cd HydraNet

pip install -r requirements.txt

# Watch the demo (no API key needed)
python battle/demo_cli.py

# Run a real battle (needs OPENAI_API_KEY)
cp .env.example .env
# Add your OPENAI_API_KEY
python main.py
# Dashboard: http://localhost:8000
```

### Run a battle from Python:

```python
from battle.arena import BattleArena, Combatant

arena = BattleArena()
arena.add_combatant(Combatant("GPT-4o", "gpt-4o", "openai"))
arena.add_combatant(Combatant("Claude", "claude-sonnet", "anthropic"))
arena.add_combatant(Combatant("Gemini", "gemini-pro", "google"))

result = await arena.battle("Write a Python web scraper with error handling")

print(f"Winner: {result.winner}")
print(f"Evolved answer:\n{result.evolved_answer}")
```

## Philosophy

> Stop asking one AI. Make them fight.

Single-AI outputs have blind spots. They hallucinate. They play it safe. There's no error correction.

HydraNet introduces **adversarial pressure** into AI generation. When AIs know their work will be critiqued, the output quality goes up. When you combine the best of 3 independent approaches, you get something no single model could produce.

This isn't prompt engineering. This is **prompt evolution**.
